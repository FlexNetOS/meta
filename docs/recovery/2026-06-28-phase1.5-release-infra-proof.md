## Envctl key request and mint contract

Filed issue: <https://github.com/FlexNetOS/envctl/issues/405>

Known available per owner:

- `CARGO_REGISTRY_TOKEN` for crates.io.
- `NPM_TOKEN` / `NODE_AUTH_TOKEN` for Node/npm/NAPI.

Still needed or needs confirmation:

| Secret / token | Purpose | Minimum scope |
| --- | --- | --- |
| `HOMEBREW_TAP_TOKEN` | Push formula branch/PRs into `FlexNetOS/homebrew-tap` | `contents:write`, `pull_requests:write`, metadata read on the tap |
| `ORG_REPO_BOOTSTRAP_TOKEN` or GitHub App installation token | Create/configure the three FlexNetOS release repos | one-time admin/bootstrap, then retire |
| `CORE_REPO_PAT` / FlexNetOS core source-release token | Download CLI release assets from private source if private | read releases/assets only |
| `DESKTOP_SOURCE_REPO_PAT` | Download Desktop DMG assets from private source if private | read releases/assets only |
| GitHub App app id/private key/installation id | Envctl short-lived repo-scoped token minting | broker-only in envctl vault |
| `DISCORD_RELEASES_WEBHOOK_URL` | Optional release notifications | optional webhook only |

Envctl's confirmed consumer shape:

```bash
secretctl mint-github \
  --installation-id "$INSTALLATION_ID" \
  --repository-ids "$REPO_IDS" \
  --permissions contents:write,metadata:read,pull_requests:write,actions:read \
  --ttl-secs 3600 \
  --output json
```

Expected stdout shape:

```json
{"token":"<scoped>","expires_at_unix":1234567890}
```

Do not paste secrets into issues, PRs, docs, chat, or logs. Store broker-only values in envctl/secretd and hand out only short-lived scoped tokens.
