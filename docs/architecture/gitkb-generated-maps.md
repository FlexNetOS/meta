# GitKB Generated Maps

These maps should be generated from GitKB or code intelligence rather than hand
maintained where possible.

## Generated or Queryable

- Document graph
- Task board and status views
- Ready-task ranking
- Context bundles
- Code symbols
- Callers and callees
- Impact analysis
- Entry points
- Flow queries
- Search and refs
- Stats and doctor output

## Hand-Maintained

- Glossary
- Human-facing architecture explanation
- Evidence appendix
- Decision notes and open questions

## Rule

If a GitKB command can prove it, prefer that output over a static copy in docs.
