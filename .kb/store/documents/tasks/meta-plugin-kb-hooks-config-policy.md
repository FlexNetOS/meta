# Summary

Model GitKB hook toggles and assistant hook behavior as `meta-plugin` policy, generated into harness adapters only after live command support is proven.

# Source Evidence

- `docs/gitkb/reference-configuration.md` defines `[hooks]` toggles in `.kb/config.toml`.
- `docs/gitkb/cli-reference.md` documents `git-kb init git` for git hooks such as reference-transaction branch symbol pruning.
- Earlier proof showed `git-kb hook` was not a live command in this environment, so hook adapters must not call undocumented commands.

# Commands And Configs Covered

- `git-kb init git`
- `.kb/config.toml`
- `[hooks]`
- `context_injection`
- `auto_commit_link`
- `auto_worktree_bind`
- `auto_progress`
- `prompt_context`
- Git hook files generated by GitKB or meta.
- Assistant hook files generated by Claude/Codex/plugin adapters.

# Acceptance Criteria

- [x] Treat `.kb/config.toml [hooks]` as policy input, not as an instruction to invent assistant hooks.
- [x] Verify live `git-kb init git --help` and generated output before changing git hooks.
- [x] Verify there is no required `git-kb hook` command before generating any assistant hook that calls it.
- [x] Define how `context_injection`, `auto_commit_link`, `auto_worktree_bind`, `auto_progress`, and `prompt_context` map to Claude/Codex behavior.
- [x] Add validation that fails when generated assistant hooks call missing commands.
- [x] Add validation that stale trusted hook state is reported before mutation.
- [x] Destructive hook cleanup requires backup and exact path match.

# Policy

- `.kb/config.toml [hooks]` is policy input only. Missing hook keys are not permission to invent assistant hooks.
- `context_injection` and `prompt_context` map to context loading/injection behavior, not shell cleanup.
- `auto_commit_link`, `auto_worktree_bind`, and `auto_progress` map to explicit GitKB workflow actions only after live command proof and user-approved adapter generation.
- Generated assistant hooks must not call nonexistent commands. `meta plugin doctor-hooks` validates this rule against assistant hook JSON surfaces.
- Destructive hook cleanup requires backup, exact path match, and explicit approval.
